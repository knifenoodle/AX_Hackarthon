## 1. 결과물 임팩트 논의

### Q. output이 약하다는 고민 — 구체화 또는 대안 아이디어?

**현황 문제**: "VoC 분류·인사이트 리포트"가 1회성 문서로 끝나, AI 활용 역량이 잘 드러나지 않음

**브레인스토밍 결과 — 세 가지 방향 검토**

| 옵션 | 설명 | 결론 |
|------|------|------|
| A. 자동 분류 파이썬 스크립트 | `python classify_voc.py voc.csv` → 분류표 + 통계 자동 출력 | Standard 레벨로 포함 |
| B. Streamlit VoC 대시보드 | `streamlit run dashboard.py` 로컬 실행, 브라우저 시연 | **제외 범위**로 결정 (노션 문서 ⑥항) |
| C. Skill 자동화 패키지 | `/명령` 시퀀스로 재현 가능한 파이프라인 | Standard 레벨로 포함 |

## 2. 레벨 구조 설계

### Q. 현재 목표를 레벨 1로 하고 레벨 2, 3을 설정한다면?

**설계 원칙**: "1회성 결과물" → "매주 돌아가는 자동화"로 발전하는 누적형 구조

### 최종 확정 레벨

```
🟢 Basic (100점)  — 분석가 수준
  → VoC 44건을 4유형 분류 + 건수·비율 집계 + 탄소규제 인사이트 3개
  → 제출: 노션 또는 구글 독스 공개 링크

🟡 Standard (+30점, 누적 130점)  — PM 수준
  → Basic + 재사용 가능한 파이프라인
  → 새 CSV 넣으면 정제→분류→집계→리포트 자동 생성 (/명령 또는 스크립트)
  → 주간 리포트 섹션·지표·형식을 본인이 직접 설계

🔴 Challenge (+30점, 누적 160점)  — 시니어 PM 수준
  → Standard + VoC 패턴에서 제품 개선 기획안 자동 생성
  → 기획안 구성: 문제 정의 / 근거 VoC / 개선 제안 / 우선순위
  → 기획안 양식·우선순위 기준도 본인이 설계
```

## 3. CSV 데이터 재설계

### Q. 노션 문서(Problem 01 v2)와 로컬 파일 정합성 맞추기

**변경 전 컬럼** (구버전)
```
id, 날짜, 채널, 고객명, 고객유형, VoC내용, 유형, 처리상태
```

**변경 후 컬럼** (노션 v2 기준)
```
id, date, channel, customer_type, content, keyword_hint
```

**주요 변경 사항**
- `고객명` 제거 (개인정보, 분류에 불필요)
- `유형` 제거 (참여자가 직접 분류해야 하므로 정답 노출 금지)
- `처리상태` 제거 (문제 스코프 외)
- `keyword_hint` 추가 (도메인 이해 보조)
- `customer_type` 값 변경: `제조업/금융/공공/유통` → `1차 협력사/2차 협력사/OEM/기타`
- `channel` 값 변경: `설문→이메일`, `미팅→미팅메모`

**의도적 데이터 품질 이슈 (참여자가 처리해야 할 과제)**
- 날짜 형식 3종 혼용: `2026-05-02` / `2026/05/06` / `5월 4일`
- 채널 결측 2건: id 032, 035
- 중복 행 1건: id 031 = id 001 (완전 동일)
