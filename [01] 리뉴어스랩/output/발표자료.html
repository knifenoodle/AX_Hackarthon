받은 문제 (원문)

고객의 목소리(VoC)가 이메일·채널톡·설문 등으로 흩어져 들어오지만, CS팀이 수작업 엑셀로 정리해 유형별 패턴 파악·의사결정 활용이 안 된다. VoC 45건을 4유형으로 분류하고 규제(CSRD·CBAM) 맥락 인사이트 3개를 도출하라.

만든 것

고객의 목소리를 '요구한 행동' 기준으로 4유형 자동 분류하고, 규제 대응 인사이트와 '먼저 고칠 것' 우선순위까지 뽑아내는 분석 리포트 — 새 데이터로도 같은 품질로 재현되는 처리 흐름 포함.

문제 이해

고객사가 낸 문제를 제 언어로 다시 정의해, 진짜 풀어야 할 지점을 짚었습니다.

기업 원문 문제

VoC가 여러 채널로 흩어져 들어오고 CS팀이 수작업 엑셀로 정리 중. CBAM·CSRD로 문의·불만 급증. 45건을 불만·요청·칭찬·문의로 분류하고 규제 맥락 인사이트 3개를 도출.

내가 재정의한 문제

진짜 문제는 '한 번 분류'가 아니라 'PM인 내가 감이 아니라, 매출 기여도까지 반영한 반복 가능한 기준으로 우선순위를 방어하는 것'. 어느 등급 고객의 이슈인지까지 나와야 로드맵 우선순위를 근거로 설명할 수 있다.

문제의 크기 (Before)

담당자마다 분류 기준이 달라 매출 기여도 큰 고객(OEM)의 이슈도 다른 문의와 뒤섞여 대응이 늦어질 위험이 있다. B2B SaaS 업계 기준 엔터프라이즈 계정 가치는 SMB의 10배 이상이라, 이 등급의 대응 지연은 곧 고객 만족도 하락·재계약 이탈로 이어진다.

스코프 — 푼 것 / 안 푼 것

○ 푼 것 — 분류·집계·규제 인사이트·재현 흐름·제품 우선순위 기획안
✕ 안 푼 것 — 실시간 채널 연동·대시보드(과제 제약 → 정적 리포트로 대체)

접근 가설 / 핵심 흐름

말투로 분류하면 사람마다 결과가 달라진다 → '고객이 실제로 무엇을 요구했는가(행동)'를 유일 기준으로 삼았다.

데이터 정리 → 4유형 분류 → 건수·비율 집계 → 규제 맥락 해석 → 우선순위

같은 원칙을 우선순위에도 적용 — 감이 아니라 반복 빈도·이탈 위험·매출 규모 3축 점수로 계산해 재현 가능하게 고정했다.

버린 대안 & 이유

① 한 목소리에 꼬리표 여러 개 → 합계가 100%를 넘어 집계가 깨짐 → 대표 유형 1개+부가 신호로 분리
② 말투·긴급도 기준 분류 → 판정자마다 흔들려 기각

★ 막혔던 지점 & 돌파 방법

'수치가 30% 틀렸다'며 화난 고객 — 불만인가, 문의인가? 말투는 불만이지만 실제 요구는 '근거 설명'이었다. → '말투가 아니라 요구한 행동으로 판정' 규칙을 고정. 덕분에 독립적으로 다시 분류해도 44건 전부 같은 결과가 나왔다.

AI vs 내 판단

AI = 45건을 읽고 분류 초안·집계·문서화 실행. 나 = 분류 '기준'·경계 케이스 규칙·우선순위 점수 설계. 기준은 내가, 실행은 AI가.
