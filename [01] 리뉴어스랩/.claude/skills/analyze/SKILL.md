# /analyze — VoC 데이터 파악·품질 점검

VoC CSV를 분류하기 **전에** 데이터의 상태를 진단하고 정제 계획을 세우는 단계다.
이 단계에서는 **데이터를 수정하지 않는다.** 진단 결과와 "정제 계획"만 출력한다.

## 입력
- `data/voc.csv` (기본). 다른 파일이면 사용자가 경로를 지정한다.
- 기대 스키마: `id, date, channel, customer_type, content, keyword_hint`
  - 컬럼명이 다르면 의미 기준으로 매핑하고, 매핑 결과를 명시한다.

## 절차
1. **로드·개요**: 전체 행 수, 컬럼 목록, 앞 3행 샘플을 출력한다.
2. **결측 점검**: 컬럼별 빈 셀 개수와 해당 `id`를 나열한다. (특히 `channel`, `content`)
3. **중복 점검**:
   - `content`가 완전히 동일한 행 그룹을 찾는다.
   - 완전 동일 행(내용·날짜·채널 모두 같음)은 **중복 후보**로 표시하고, 남길 대표 1건과 제거 대상 `id`를 제안한다.
   - 내용은 유사하나 날짜/채널이 다르면 "반복 재유입"일 수 있으니 제거하지 말고 별도 표시한다.
4. **날짜 형식 점검**: 등장하는 날짜 표기 형식을 모두 식별한다(예: `YYYY-MM-DD`, `YYYY/MM/DD`, `M월 D일`). 각 형식의 건수와 ISO 변환 시 주의점(연도 결측 등)을 기록한다.
5. **빈 내용 점검**: `content`가 비었거나 공백뿐인 행을 찾는다. → 분류 불가이므로 제외 후보로 표시.
6. **분포 미리보기**: `keyword_hint`(있으면) 값 분포를 세어 대략적 유형 감을 잡는다. ※ 이 값은 참고용일 뿐 분류 근거가 아니다(분류는 /insight의 CTA 규칙으로 판정).
7. **정제 계획 제안**: 아래 형식으로 요약한다.

## 정제 규칙 (기본값 — 새 데이터에도 동일 적용)
- **완전 동일 중복** → 대표 1건 남기고 제거. 집계 분모는 제거 후 건수.
- **날짜** → ISO(`YYYY-MM-DD`)로 통일. `M월 D일`은 데이터 기간의 연도를 채운다(불확실하면 사용자에게 확인).
- **채널 결측** → 임의 추정 금지, 결측 유지(내용 분류에는 영향 없음).
- **빈 내용** → 분류 불가로 제외하고, 제외 건수를 리포트에 명시.

## 출력 형식
```
## 데이터 개요
- 총 N행 / 컬럼: ...
## 품질 이슈
- 결측: channel M건(id ...), content K건(id ...)
- 중복: id X = id Y 완전 동일 → X 제거 제안
- 날짜 형식: A형식 n건 / B형식 m건 / C형식 k건
## 정제 계획 (요약)
- (위 규칙에 따른 처리 항목 나열, 처리 후 예상 건수)
```

## 판단이 필요할 때
새 데이터에서 위 규칙으로 자동 처리하기 애매한 경우(예: 유사하지만 완전 동일하지 않은 중복, 연도 판별 불가한 날짜)는 **임의로 처리하지 말고** 사용자에게 옵션과 함께 질문하고, 결정은 `decisions.md`에 기록한다.
