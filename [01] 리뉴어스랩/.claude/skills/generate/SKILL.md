# /generate — 제출물 자동 생성

`/insight`가 만든 `output/voc-classification.csv`를 단일 진실 원천으로 삼아 제출물을 렌더링한다.
숫자는 CSV에서 **다시 집계**해 채운다(하드코딩 금지). CSV가 바뀌면 리포트도 그대로 갱신된다.

## 산출물
1. **output/voc-report.md** — 제출용(노션/구글독스 붙여넣기). `output/template.md` 구조를 따른다. [Standard] 파이프라인·주간 리포트 설계 섹션 포함.
2. **output/voc-report.html** — 시각 리포트(정적, 인라인 CSS/JS, 외부 의존성 없음). Basic 시각 리포트(분포·분류표·인사이트) **+ 하단에 주간 VoC 리포트 섹션**을 한 파일에 함께 담는다. 파이프라인 재현성 설계 서술은 넣지 않는다(md 전용).

## voc-report.md 구성 (template.md 기준)
1. **데이터 개요**: 분석 대상·기간·정제 내용·분류 방법 요약.
2. **유형별 건수·비율**: 4유형 표 + 합계 100%. (CSV에서 primary_type 카운트)
3. **VoC 유형별 분류표**: 전 행. 컬럼 = `id | date | customer_type | content 요약 | 유형 | 부가 신호`.
   - "부가 신호"에는 secondary_aspects / 긴급 / 질문N을 사람이 읽기 쉬운 한국어로 표기. 없으면 `—`.
4. **탄소규제 맥락 인사이트 3개**: 내용·근거·비즈니스 의미.
5. **의사결정 로그 요약**: `decisions.md` 핵심을 표로 압축.

## voc-report.html 스타일 규칙 (현재 리포트와 일관 유지)
- **이모지 사용 금지.** 레벨 배지("Basic/Standard 몇 점") 넣지 않음. 제목은 `VoC 분류 인사이트 리포트`.
- 카드/배지 색상 규칙: 불만=적색, 기능 요청=청색, 칭찬=녹색, 일반 문의=회색.
- 유형별 막대 그래프 + 보조 지표 칩(긴급·다중질문·부가 신호) + 필터(전체/4유형/긴급/다중질문) 가능한 분류표.
- 반응형(모바일에서 표는 가로 스크롤).

## 집계 방법
- **비율** = 유형 건수 / 정제 후 총건수 × 100, 소수 첫째 자리.
- **긴급/다중질문**은 별도 지표로 표기(유형 분포에 합산하지 않음).

## 주간 리포트 생성 규칙
`date`로 주(월~일 또는 데이터 시작일 기준 7일 단위)를 나누고, 각 주마다:
- **KPI 5종**: 전체 VoC / 일반 문의 / 기능 요청 / 칭찬 / 불만 — 각각 **전주 대비 증감(WoW)** 병기.
- **해결율**(KPI 하단): 처리상태 '완료' ÷ 전체 VoC (+WoW). CS 처리상태 필드(신규 → 진행중 → 완료)로 산출. 원본에 처리상태 컬럼이 없으면 산출식을 명시하고 예시값임을 표기. 전체 해결율 아래 **유형별 해결율 4개(일반 문의·기능 요청·칭찬·불만)**를 카드로 병기.
- **즉시 대응 목록**: 이번 주 `불만` + `urgency=높음` 행. 컬럼 = id·customer_type·내용·긴급도·권장 대응(SLA). 긴급=24h, 중간=3영업일 기준.
- **반복 요청 TOP**: primary/secondary의 기능 요청을 유사 주제로 묶어 **누적 빈도 상위**를 표기(1회성보다 반복 우선). 대표 id 인용.
- **탄소규제 신규 패턴**: 이번 주 CBAM/CSRD/계산정확성 등 규제 맥락 신호를 2~3개 요약.
- 섹션 제목 옆 팀 라벨(CS팀/제품팀)은 표기하지 않는다. 팀 매핑은 설계 근거표의 '사용팀' 열로만 문서화한다.
- 매주 **동일 구조**로 생성해 시계열 비교가 가능하게 한다.
- **시각화**: 주간 리포트는 `voc-report.html` 하단에 구분선(week-divider) 뒤로 섹션을 추가해 렌더링한다(별도 파일 아님). KPI는 5장 카드(전체/문의/기능/칭찬/불만 + WoW 화살표), 해결율은 막대 + 전주 대비 %p, 즉시 대응·반복 요청은 표, 규제 패턴은 카드. 색상 규칙은 Basic 리포트와 동일(불만=적색, 기능 요청=청색, 칭찬=녹색, 일반 문의=회색), 이모지 금지.

세부 설계 근거(왜 이 섹션·지표인가, CS·제품팀 관점)는 voc-report.md의 [Standard] 섹션에 서술한다.
